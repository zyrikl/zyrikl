Zyrikl Tutorial

Zyrikl is a project created by Charles Wang. It makes coding a website much, much easier.

Starting your first app

Zyrikl can be used anywhere. In your HTML file, type in:

<zyrikl>
    echo "Hello, World";
</zyrikl>
<script src="https://cdn.jsdelivr.net/gh/zyrikl/zyrikl/source.js"></script>

View the document. It should show a webpage with the words "Hello, World" on it. Now, onto the actual code!

Zyrikl Documentation

Builders

Builders are functions that add blocks of content onto your website. They are described using line IDs with the following syntax: #line4 for line 4. Zyrikl is content-based, so any line in the Zyrikl code gets parsed to the compiler. The Zyrikl Development Team does not recommend single line code, due to our unique styling method.

echo

Displays a paragraph. Can be used with BEGIN "echo" and END "echo".

bold

Bolds text. Does not end with a newline.

italics

Italicizes text. Does not end with a newline.

print

Displays unformatted text with a newline.

line

Displays unformatted text without a newline.

title

Displays large font text. Has BEGIN/END attributes.

header1

Displays large font text. Has BEGIN/END attributes.

header2

Displays medium font text. Has BEGIN/END attributes.

header3

Displays medium font text. Has BEGIN/END attributes.

header4

Displays heading font text. Has BEGIN/END attributes.

header5

Displays heading font text. Has BEGIN/END attributes.

list_entry

Can be used with unordered and ordered to create a list entry.

link

Takes two arguments. Example syntax:

link "https://google.com" "This is Google";
image

Displays an image.

webdriver

Displays another website

line

Displays a line across the screen.

break

Displays a space in between lines.

button

Shows a button.

input_text

Creates an input that takes text.

input_password

Creates an input that takes a password.

submit_button

Shows a button that says "Submit".

write

Creates a larger version of input_text.

mono

Creates unformatted text.

Systematics

Systematics change the way a builder works, or edit the physical page itself. They are always capitalized. They can be used to make builders more useful, such as wrapping a section of content in a page. Systematics are also used to style and add interactive function.

HEAD

Changes tab name in browser.

BEGIN & END

Starts and ends an element. Can have the following arguments:

  • echo
  • title
  • header1
  • header2
  • header3
  • header4
  • header5
  • unordered
  • ordered
  • container
  • code_block
  • button
  • link (requires link after BEGIN attribute, but END attribute will not take it.)

    • Find the references above.

    IMPORT

    Currently takes 3 arguments: "stellar", "retro", and "docs".
